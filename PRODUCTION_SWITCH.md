# 🚀 本番モード切替手順

「お花屋さんが本物のお金を受け取る準備ができた」ときの切替手順です。
今はテストモード（`sk_test_`）で開発・確認できる状態。

## いつ切り替える？

以下が **すべて** 揃ったタイミング:

- [ ] お花屋さん（連結先テナント）が本人確認書類を提出済み
- [ ] お花屋さんが Stripe からの審査を通過済み
- [ ] お花屋さんの銀行口座が登録済み（振込先）
- [ ] テストモードで一連の動作確認完了
- [ ] 注文確認メールが届くことを確認
- [ ] 利用規約・特定商取引法のページが用意できている

## 切替手順

### 1️⃣ Stripe ダッシュボードを Live モードに切替

1. https://dashboard.stripe.com を開く
2. 左下のトグルで **「テストモード」→「ライブ」** に切替
3. 初回はライブ用の本人確認を求められる場合あり（プラットフォームの情報入力）

### 2️⃣ Live モードの API キー取得

ライブ環境で：

- 開発者 → APIキー
  - 公開可能キー (`pk_live_...`) → コピー
  - シークレットキー (`sk_live_...`) → コピー
- 開発者 → Connect → 設定
  - Client ID (`ca_...`) → コピー（ライブ用）

### 3️⃣ Live モードの Webhook 再作成

1. ライブ環境で 開発者 → Webhooks → 「+ エンドポイントを追加」
2. URL: `https://www.noodleflorix.com/api/stripe/webhook`
3. イベント:
   - `checkout.session.completed`
   - `payment_intent.payment_failed`
   - `charge.refunded`
   - `account.updated`
4. 作成後、**署名シークレット** (`whsec_...`) をコピー

### 4️⃣ Vercel の環境変数を Live キーに更新

Vercel → Settings → Environment Variables で以下を **編集**：

| Key | 旧（テスト）| 新（本番） |
|---|---|---|
| `STRIPE_SECRET_KEY` | `sk_test_...` | `sk_live_...` |
| `STRIPE_PUBLISHABLE_KEY` | `pk_test_...` | `pk_live_...` |
| `STRIPE_CLIENT_ID` | `ca_xxxx`（テスト用）| `ca_xxxx`（ライブ用、新規） |
| `STRIPE_WEBHOOK_SECRET` | `whsec_xxx`（テスト/ローカル）| `whsec_xxx`（ライブ） |

### 5️⃣ 再デプロイ

Vercel → Deployments → 最新 → 「Redeploy」

### 6️⃣ お花屋さんを Live モードで再オンボーディング

⚠️ **重要**: テストモードで連携したアカウントは Live モードでは使えません。

1. しーちゃんのアプリで `/staff/settings` → 決済設定タブ
2. **「Stripe接続を解除する」** をクリック（既存テスト接続をクリア）
3. **「新規でStripe決済を有効化」** を再度実行
4. お花屋さんに **本物の事業情報・銀行口座** を入力してもらう
5. Stripe審査通過 → 接続完了

### 7️⃣ 動作確認

- 1円とか少額で実カード決済テスト（自分でやる）
- 決済成功 → スタッフ画面で受注確認
- お花屋さんの口座に振込予定が登録されてるか Stripe ダッシュボードで確認
- 必要なら自分にメールで送って動作確認
- ⚠️ テストカード `4242 4242...` は **使えなくなる** ので注意

## ロールバック（万が一）

何か問題が起きたら：

1. Vercel環境変数を **テストキー** に戻す
2. 再デプロイ
3. Stripeダッシュボードを **テストモード** に戻す

すぐに開発環境に戻せます。

## 注意事項

- 一度本番モードで決済を受け始めたら、テストモードに戻す時はデータが混在しないよう注意
- お客様への請求書発行のためにも本物の事業情報・連絡先が必要
- 特定商取引法に基づく表記ページが必要（法律上）
