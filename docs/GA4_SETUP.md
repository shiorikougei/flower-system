# Google Analytics 4（GA4）セットアップ手順

> [SEO-#14] アクセス解析を導入してデータドリブンな運用を始める

---

## なぜ必要か

- どのページがどれだけ見られてるか把握
- 検索キーワード分析（Search Consoleと連携）
- コンバージョン（注文・見積依頼）の追跡
- ユーザー属性分析
- **無料・15分でセットアップ**

---

## セットアップ手順

### Step 1: GA4 アカウント作成（5分）

1. https://analytics.google.com/ にアクセス
2. **「測定を開始」** をクリック
3. アカウント名: `FLORIX`（任意）
4. プロパティ名: `FLORIX 本番`（任意）
5. 業種・規模: 適切なものを選択
6. 「ビジネス目標」: **「リードの生成」「販売の促進」** を選ぶ
7. 「データの収集」: **「ウェブ」** を選ぶ
8. ウェブサイトURL: `https://www.noodleflorix.com`
9. ストリーム名: 任意

### Step 2: 測定ID を取得

ストリーム作成後、画面に表示される **「測定ID」** をコピー:

```
G-XXXXXXXXXX
```

⚠️ この ID を後で使う！

### Step 3: Vercel に環境変数を追加

1. Vercel ダッシュボード → **flower-system** → **Settings** → **Environment Variables**
2. **「Add New」**
3. 入力:

```
Key:   NEXT_PUBLIC_GA_ID
Value: G-XXXXXXXXXX  ← Step 2 でコピーした測定ID
```

4. **Production / Preview / Development** すべてにチェック
5. **「Save」**

### Step 4: 再デプロイ

環境変数の追加は再デプロイで反映:

```bash
# または Vercel ダッシュボード → Deployments → 最新の「⋯」 → Redeploy
```

## ✅ 動作確認

### 1. ページにアクセス

https://www.noodleflorix.com を開いて、デベロッパーツールの **Network** タブで `gtag/js` のリクエストが飛んでいることを確認

### 2. GA4 のリアルタイムレポート

GA4 ダッシュボード → **左メニュー「レポート」** → **「リアルタイム」**

→ 自分のアクセスが反映されればOK✨ （1〜3分かかる場合あり）

## プライバシー配慮

実装済みの保護機能:

| 項目 | 設定 | 効果 |
|------|------|------|
| **IP匿名化** | `anonymize_ip: true` | IPの最後のオクテットを匿名化 |
| **Cookie寿命** | 90日に短縮 | デフォルト2年→90日 |
| **Googleシグナル** | OFF | 広告連動の個人識別なし |
| **管理画面除外** | /staff /owner /api | 管理者の自分のアクセスはカウントしない |

---

## 重要イベントの設定（推奨）

GA4 ダッシュボード → **管理** → **イベント** で以下を「キーイベント」に設定:

| イベント名 | 意味 |
|-----------|------|
| `page_view` | ページ表示（自動） |
| `view_item` | 商品ページ表示（自動・推奨：商品ページに自動送信） |
| `add_to_cart` | カート追加 |
| `purchase` | 注文完了 |
| `generate_lead` | 見積依頼送信 |

これらのイベントが計測されることで、コンバージョン率の改善が可能になります。

---

## Search Console との連携（強く推奨）

1. GA4 → **管理** → **Search Console のリンク設定**
2. **「リンク」** をクリック
3. 既に登録済みの `https://www.noodleflorix.com` を選択
4. 連携完了 ✨

連携後、GA4 から検索クエリ・順位・クリック数が見れる:
- **「集客」→「Search Console」→「クエリ」**
- どんなキーワードで来訪されているか可視化

## トラブルシューティング

### 「リアルタイムでデータが出ない」

- 環境変数の `NEXT_PUBLIC_GA_ID` が正しく設定されているか確認
- 再デプロイ済みか確認
- ブラウザの広告ブロッカー（uBlock等）を無効化して再アクセス
- 数分待つ

### 「測定IDが間違っているか不明」

GA4 → **管理** → **データストリーム** → 該当ストリーム → 「測定ID」を再確認
