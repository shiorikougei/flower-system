# LINE公式アカウント 運用方針マニュアル

> 各モニター店舗向けの LINE公式アカウント設定・運用ガイド
> 最終更新: 2026年5月

---

## 🎯 基本方針

**「自動応答してます感」を出さない、人が対応してる感を保つ**

| やる | やらない |
|---|---|
| ✅ 注文確認・入金・完成写真など業務通知の自動送信 | ❌ お客様の質問に対する自動応答 |
| ✅ リッチメニューでよくある質問は自己解決 | ❌ 「ありがとうございます」だけのテンプレ返信 |
| ✅ お客様の質問は店舗の人が直接返信 | ❌ AI応答メッセージ |
| ✅ 受電・電話相談もOK | ❌ 即レス重視（5〜15分遅れてOK、内容重視） |

## 📋 LINE公式アカウントマネージャー 必須設定

設定先: https://manager.line.biz/

### A. 応答設定

| 項目 | 設定値 |
|---|---|
| 応答モード | **Bot** |
| 応答メッセージ | **OFF** ⚠️（テンプレ自動応答は使わない） |
| AI応答メッセージ | **OFF** |
| Webhook | **ON**（このアプリのWebhookを使用） |
| あいさつメッセージ | **ON**（次項のカスタム文面） |

### B. あいさつメッセージ（友達追加時）

> ※既にこのアプリのWebhookでも友達追加メッセージを送る実装があるため、LINE公式の「あいさつメッセージ」はOFFにしても良い。両方ONだと2通届くので注意。

**Webhook側のあいさつ文面（実装済）**:
```
○○さん、友だち追加ありがとうございます🌸

こちらでご注文の進捗をお届けするには、ご注文時にご登録いただいた
【メールアドレス】を このトークに送信してください。

例: example@gmail.com
```

**カスタマイズしたい場合**: `app/api/line/webhook/[tenantId]/route.js` の `replyText` 部分を編集

### C. Webhook URL設定

LINE Developers Console → Messaging API設定 → Webhook URL に以下を設定:

```
https://noodleflorix.com/api/line/webhook/{tenantId}
```

`{tenantId}` は各店舗のID（設定 → LINE連携タブで自動表示・コピーボタンあり）。

## 🎨 リッチメニュー構成（推奨6分割）

LINE公式アカウントマネージャー → リッチメニュー → 新規作成

| 配置 | アイコン | ボタン名 | 動作タイプ | 設定内容 |
|---|---|---|---|---|
| 左上 | 🌷 | ご注文 | リンク | `https://noodleflorix.com/order/{tenantId}/{shopId}` |
| 中央上 | 💐 | 商品カタログ | リンク | `https://noodleflorix.com/order/{tenantId}/{shopId}/shop` |
| 右上 | 📋 | 注文履歴・マイページ | リンク | `https://noodleflorix.com/order/{tenantId}/{shopId}/history` |
| 左下 | 🕒 | 営業時間・アクセス | テキストメッセージ送信 | キーワード「営業時間」を送信 |
| 中央下 | 📞 | 電話する | リンク | `tel:011-600-1878`（店舗番号） |
| 右下 | ❓ | よくあるご質問 | テキストメッセージ送信 | キーワード「FAQ」を送信 |

> 💡 リッチメニュー画像は LINE公式のテンプレート or Canva で作成可能。サイズ: 2500×1686px (大) / 2500×843px (小)

## 💬 キーワード自動応答（任意設定）

LINE公式アカウントマネージャー → 自動応答メッセージ → キーワード応答

「会話っぽい質問」には使わず、**事実情報の自動返答だけ** に限定:

| キーワード | 応答内容 | 注意点 |
|---|---|---|
| 営業時間 | 営業時間 + 定休日 + 各店舗別の差異 | 単純な事実なのでOK |
| アクセス・住所 | 店舗住所 + Google Mapsリンク | 同上 |
| 配送料 | 配送エリアと料金表 | 複雑な相談はNG |
| 駐車場 | 駐車場情報 | 同上 |
| 名刺・領収書 | 「マイページから領収書発行できます https://...」 | リンク誘導 |

**自動応答してはいけないキーワード例**（必ず人が返信）:
- 「明日の14時にバラ買えますか？」（在庫・予約相談）
- 「予算1万円で何作れる？」（個別提案）
- 「キャンセルできますか？」（個別判断）
- クレーム・苦情系全般

## 🌙 営業時間外の対応

LINE公式アカウントマネージャー → 応答設定 → 「応答時間」設定

### 営業時間内
通常の応答（人が対応）

### 営業時間外
以下のような自動メッセージを設定:

```
お問い合わせありがとうございます🌸
ただいま営業時間外のため、すぐにお返事できません。

✓ ご注文は24時間オンラインで承っております
　https://noodleflorix.com/order/{tenantId}/{shopId}

✓ 急ぎのお電話は翌営業日 10:00 以降にお願いいたします

通常、翌営業日中にこちらのトークでお返事いたします。
お急ぎの方はお電話でお問い合わせくださいませ🙇‍♀️
```

## 🤖 アプリから自動送信される業務通知（実装済み）

これらは「業務通知」として明確なので自動でも違和感なし:

| トリガー | 内容 | タイミング |
|---|---|---|
| 注文受付 | 注文確認・お支払い・納品予定日 | お客様注文確定時 |
| 入金確認 | 銀行振込の入金確認・確定納品日 | スタッフが「入金済」操作時 |
| 完成写真 | 完成品の写真 + メッセージ | スタッフが写真アップ時 |
| お渡し・配達完了 | 完了通知（手動送信） | スタッフが手動でメール送信時 |
| 入荷通知 | 在庫切れ商品が入荷した時 | スタッフが「入荷通知送信」操作時 |
| 記念日リマインダー | 登録された記念日の1週間前 | 毎日朝9:00 自動チェック |

すべて **メールと同じ内容がLINEにも届く**（LINE連携が有効な場合）。

## 📞 受電対応との連携

LINE上で答えづらい質問は無理せず電話誘導:

```
お問い合わせありがとうございます。
こちらの内容、お電話でご相談させていただいた方がスムーズかと思います。

お手数ですが、下記までお電話いただけますか？
TEL: 011-600-1878（営業時間: 10:00-19:00）
```

## ✨ 「自動応答してます感」を消すための心構え

1. **テンプレ感を出さない**
   - ❌ 「お問い合わせありがとうございます」だけ
   - ✅ 「○○様、お問い合わせありがとうございます。バラのアレンジについてですね」（質問内容に触れる）

2. **スタッフ名を入れる**
   - ✅ 「(店舗名)の○○です」「担当の○○がご案内します」

3. **即レスより内容重視**
   - 5〜15分かかってもOK、ちゃんと読んで返信

4. **顧客カルテを活用**（次フェーズで実装予定）
   - 過去注文・好みを確認してから返信
   - 「いつもピンク系をお選びですよね」など個別性を出す

5. **絵文字・改行で読みやすく**
   - LINEは短い文章を複数回送る方が自然

## 🧪 連携テスト手順（店舗側で実施）

1. **アプリ側設定**
   - 設定 → LINE連携タブ
   - Channel Access Token / Channel Secret / Channel ID / 友達追加URL を入力
   - Webhook URL（青枠）をコピー

2. **LINE Developers Console 設定**
   - Messaging API設定 → Webhook URL に上記URLをペースト
   - 「Webhookの利用」を ON
   - 「応答メッセージ」を OFF

3. **アプリで「LINE連携を有効化」をON → 保存**

4. **テスト**
   - 自分の個人LINEで店舗の公式アカウントを友達追加
   - Webhookからのあいさつメッセージが届くことを確認
   - メールアドレスを送信 → 連携完了通知が届くことを確認
   - テスト注文 → LINEに注文確認通知が届くことを確認

## 🔧 トラブルシューティング

| 症状 | 原因 | 対処 |
|---|---|---|
| Webhook が届かない | Channel Secret が違う | LINE Developers で再確認 |
| あいさつメッセージが2通届く | LINE公式マネージャーのあいさつもON | 公式側のあいさつをOFFにする |
| 通知がメールのみでLINEに来ない | お客様がまだメアド送信してない | 友達追加 + メアド送信を案内 |
| 「メアドが見つからない」エラー | お客様のメアドと注文時のメアドが違う | 注文時のメアドを再確認 |
| 旧LINEに通知が届く | 自動無効化のタイミング差 | お客様にもう一度新LINEからメアド送信してもらう |

## 📝 各店舗オリジナルの設定例

### 店舗A: 個人花屋（少人数）
- リッチメニュー: シンプル4分割（ご注文/履歴/電話/営業時間）
- キーワード応答: 営業時間のみ
- 営業時間外: 翌日返信を案内

### 店舗B: 法人取引メイン
- リッチメニュー: 法人窓口/個人窓口/見積依頼/電話
- キーワード応答: 見積りフォームURL
- 営業時間外: 翌営業日対応を案内
