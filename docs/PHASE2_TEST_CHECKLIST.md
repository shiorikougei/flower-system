## 🔐 Phase 1: セキュリティ系

### #99 PII保護（エラーレスポンス）
- [ ] 見積依頼フォームでわざと不正な入力（メアド形式違反）→ エラーメッセージが「見積依頼の登録に失敗しました」と一般的になっている
- [ ] DBエラー詳細やテーブル名がレスポンスに含まれていない

### #100 PINブルートフォース対策
- [ ] 設定 → スタッフ管理 → PIN必須 ON
- [ ] 一旦ログアウトして、スタッフ切替時にPINを **わざと連続で5回間違える**
- [ ] 「5回連続で失敗しました。5分間ロックします。」と表示される
- [ ] 5分以内に再試行すると「ロック中」エラーが出る
- [ ] 監査ログに pin_lockout イベントが記録される

### #101 XSS対策
- [ ] お客様注文フォームで名前欄に `<script>alert('xss')</script>` を入力
- [ ] 注文確認メールでスクリプトが実行されず、文字列として表示される
- [ ] 受注一覧で同様に表示が崩れない

### #102 プライバシーポリシー
- [ ] https://noodleflorix.com/privacy にアクセス
- [ ] 9セクション全て表示される
- [ ] 文字化け・レイアウト崩れがない

### #103 店舗メアド確認フロー
- [ ] 設定 → 受注通知メール設定 で **わざと違うメアド** を入れて「テスト送信」
- [ ] 入力したメアドにテストメールが届く（届かないメアドは入力ミスと判明）
- [ ] エラー時は「送信失敗しました」表示
